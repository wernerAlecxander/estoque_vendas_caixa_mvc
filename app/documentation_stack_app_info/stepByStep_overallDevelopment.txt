---------------------------FASE 1---------------------------------
Infraestrutura e Banco (Backend):
Configurar Docker: Subir o container do PostgreSQL. ✅
Modelagem com Prisma: Criar o schema.prisma e rodar pnpm prisma migrate dev. ✅
Seed do Banco: Criar um script para alimentar tabelas base (Marcas, Modelos, Usuário Admin).✅
--------------------------Fase 2-------------------------------------
Autenticação e Controle de Acesso (Login e Middleware):
PARA QUE FAZER: Para proteger os dados financeiros, de estoque e de clientes do ferro velho contra acessos não autorizados.
COMO FAZER: Utilizaremos a biblioteca NextAuth.js (Auth.js) integrada ao Prisma. Criaremos uma tela de login minimalista. O middleware.ts do Next.js interceptará todas as requisições: se o usuário tentar acessar /admin ou /dashboard sem um token válido, ele será redirecionado para /login. Se o usuário for um cliente comum, ele só terá acesso à /pagina-publica
Setup do NextAuth.js / Auth.js: Configurar a estratégia JWT e o provedor de credenciais (e-mail/senha).
Middleware: Bloquear rotas internas para usuários não autenticados.
--------------------------------Fase 3-----------------
Estrutura Base de Layout (Sidebar e Dark Mode):
PARA QUE FAZER: Para unificar a navegação interna do sistema e garantir o conforto visual do operador do ferro velho (que costuma usar o sistema o dia todo).
COMO FAZER: O sideBar.tsx lerá as rotas do seu menuData.ts. Para o Dark Mode, usaremos a estratégia de classes do Tailwind (darkMode: 'class') combinada com o next-themes. O sistema salvará a preferência do usuário no localStorage
Layout Base do Admin:
Sidebar Dinâmica: Criar o componente de navegação com os submenus colapsáveis.
Contexto de Dark Mode: Configurar o Tailwind para alternar classes com base no localStorage.
-----------------------------Fase 4---------------
Core de Negócio (Painel Administrativo (Dashboard) e Rotas Internas):
PARA QUE FAZER: Para alimentar e gerenciar o coração do ferro velho (cadastros, compras, vendas e serviços).
COMO FAZER: Criaremos rotas dinâmicas no Next.js App Router (ex: app/(dashboard)/cadastrar/clientes/page.tsx). Cada rota terá formulários em React usando react-hook-form e validação com zod, enviando os dados para o PostgreSQL através de Next.js Server Actions conectadas ao Prisma.
Cadastros Básicos: Telas de Clientes, Marcas, Modelos e Usuários.
Inventário e Compras: Fluxo de entrada de Sucatas e Peças com geração de QRCode.
Vendas e Serviços: Fluxo de orçamentos, faturamento e ordens de serviço.
Caixa e Indicadores: Telas financeiras com cálculo de fórmulas econômicas.
------------------------------FASE 5-----------------------------
Módulo de Caixa e Indicadores Econômicos:
PARA QUE FAZER: Para transformar os dados brutos de compras e vendas em inteligência financeira (Margem, Ponto de Equilíbrio, Ticket Médio).
COMO FAZER: Criaremos queries agregadas no Prisma para calcular as fórmulas em tempo real no backend e exibiremos os resultados em gráficos minimalistas usando a biblioteca recharts.

-----------------------Fase 6---------------
Página Pública de Vendas (E-commerce):
PARA QUE FAZER: Para expor as peças disponíveis na internet para clientes externos comprarem ou gerarem orçamentos sem acessar o painel interno.
COMO FAZER: Criaremos uma rota pública isolada (app/(public)/produtos/page.tsx) que lista apenas itens de peca_estoque que possuem status "disponível" e exibe as fotos vindas de peca_imagens.
Vitrine Digital: Catálogo indexável de peças avulsas disponíveis no estoque, otimizado para SEO.

---------------------------fASE 7--------------------------
Geração de QR Code e Impressão:
PARA QUE FAZER: Para colar etiquetas físicas nas peças e sucatas no pátio físico do ferro velho, agilizando a busca e a baixa no estoque.
COMO FAZER: Utilizaremos a biblioteca qrcode no Node.js para gerar os códigos contendo a URL interna da peça e criaremos uma folha de estilo CSS focada em impressão (@media print) para emitir as etiquetas perfeitamente na impressora térmica.

--------------------------FASE 8------------------------------------
Empacotamento Desktop (Electron) e Mobile (Capacitor/React Native):
PARA QUE FAZER: Para permitir o uso do sistema como um aplicativo instalado no computador e nos celulares dos funcionários que operam no pátio.
COMO FAZER: Quando o Next.js estiver pronto, configuraremos o Electron para abrir a URL do seu sistema em uma janela nativa de desktop. Para o mobile, a forma mais rápida mantendo o Next.js é usar o Capacitor para empacotar o site, ou construir um app separado em React Native consumindo a API do Next.js.


ORIENTAÇÕES EXTRAS:
--------------------------------------------------
Como e Por Que Fazer os Componentes Core:
1.Rotas (Routing):
COMO FAZER: No Next.js App Router, use pastas estruturadas. Crie um grupo de rotas para a área logada app/(admin)/ e outro para a área pública app/(public)/.
PARA QUE FAZER: Organiza o projeto de forma lógica. Isola o layout do painel administrativo (que usa Sidebar) do layout do site público (que usa um menu de e-commerce tradicional).
------------------------------
2.Login
COMO FAZER: Use a biblioteca bcrypt para criptografar senhas. Crie uma tela em app/(auth)/login/ conectada ao NextAuth.
PARA QUE FAZER: Garante que apenas funcionários autorizados acessem os dados financeiros, de compras e de clientes do ferro velho.
-------------------------------
3. Middleware
COMO FAZER: Crie um arquivo middleware.ts na raiz do projeto. Ele intercepta as requisições e checa se o token JWT do usuário existe antes de renderizar páginas dentro de /admin.
PARA QUE FAZER: Segurança absoluta. Impede que qualquer pessoa acesse o painel digitando a URL diretamente no navegador.
----------------------------
4. Painel de Admin vs. Página Pública
COMO FAZER: O Admin consome dados privados via Server Actions ou APIs protegidas. A Página Pública faz buscas otimizadas no banco buscando apenas peças com status "disponível" e exibe imagens salvando o cache para performance.
PARA QUE FAZER: O admin foca em controle operacional rígido e velocidade. A página pública foca em marketing, experiência do cliente final e indexação no Google (SEO) para vender as peças mais rápido.
-----------------------------------
5. Sidebar
COMO FAZER: Crie um componente que recebe um array de objetos com os submenus que você listou. Use o lucide-react para os ícones e controle de estado do React (useState) para abrir e fechar os grupos (ex: Cadastros, Caixa).
PARA QUE FAZER: Centraliza a navegação de mais de 40 subrotas que seu sistema possui, evitando que o usuário se perca na densidade do sistema.
-----------------------------------
6. Dark Mode
COMO FAZER: Instale a biblioteca next-themes. Envolva o layout com o ThemeProvider e configure o tailwind.config.js com darkMode: 'class'. Use uma chave no localStorage para lembrar a escolha do usuário.
PARA QUE FAZER: Funcionários de ferro velho operam o sistema o dia inteiro (8h+). O Dark Mode reduz drasticamente a fadiga visual em telas de cadastro pesadas.
